```text
# nosemgrep: <rule-id> -- <reason ≥10 chars>
# shellcheck disable=SCxxxx -- <reason>
# nosec Bxxx -- <reason>
```

## Grandfathered findings (extracted-block jobs)

The `shellcheck-extracted` and `bandit-extracted` jobs run the P2.1 extractor over every `.md` in `skills/`, `agents/`, `commands/`, `templates/`, `docs/` and lint the embedded fenced code blocks as standalone files. The current baseline produces:

- **shellcheck-extracted: 23 warnings + 5 errors** across ~10 files
- **bandit-extracted: 0 findings** (1 file with AST parse error, gracefully skipped)

The ALL of the shellcheck-extracted findings come from **illustrative documentation snippets** — bash blocks that contain placeholder syntax (`<HOST_LIST>`, `<commit-sha>`, `<validator>`) or "what NOT to do" code paths that are intentionally invalid. These are documentation, not executable scripts.

Categorical breakdown:

| Code   | Severity | Pattern                                                            | Disposition |
|--------|----------|--------------------------------------------------------------------|-------------|
| SC1058 | error    | Expected 'do' (placeholder syntax in for-loops)                    | grandfather |
| SC1072 | error    | Generic "fix problems" follow-up to other parse errors             | grandfather |
| SC1073 | error    | Couldn't parse redirection (placeholder `<sha>`, `<validator>`)    | grandfather |
| SC2168 | error    | `local` outside function (intentional pitfall demo)                | grandfather |
| SC2259 | error    | Heredoc piped-input pitfall demo                                   | grandfather |
| SC2154 | warning  | Variable referenced but not assigned (snippet shows usage only)    | grandfather |
| SC2164 | warning  | `cd` without `\|\| exit` (illustrative one-liner)                   | grandfather |

**Decision:** mark both extracted jobs as `continue-on-error: true` (informational) for the duration of the baseline window. The jobs still run and surface findings in the CI log; they do not block PRs.
